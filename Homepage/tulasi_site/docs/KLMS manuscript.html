Data structures in the cell-probe model

[!def] Static Data Structure Problem (Static DSP) A Static Data Structure Problem is given by the tuple (D,Q,A,f) where - D is the domain of possible data, - Q is the domain of possible queries, - A is the set of possible answers and - f: Q \times D \to A is a map where f(x, y) is the answer to query x about data y. ^920054

The Cell Probe Model defined by Yao ~ is a deterministic model for Data Structures useful is proving lower bounds. In this model, the data is preprocessed and stored in s memory cells each capable of holding words of w bits. Each query is modelled as a decision tree which in each time step probes a memory location and based on the answer decides which location to probe next.

A solution with parameters s, b, and t is a method of storing any y \in D as a data structure \phi(y) in the memory of a random access machine, using s memory cells, each containing w bits, so that any query in Q can be answered by accessing at most t memory cells.

[!def] Solution/ Scheme (s,w,t) A (s,w,t) Solution to a Static Data Structure Problem (D,Q,f) is given by the tuple (s,t,\phi,\{Q_i\}) where - \phi: D \to [2^w]^s is an encoding map - Q_i is a decision tree of depth at most t for each query i \in Q. ^873ea3

Let P = (D,Q,f) be a DSP. Under some fixed ordering the Data Domain D and Query Domain, each Static DSP can be represented using a table called the whose rows are labelled by the queries from Q and the columns are labelled by the data points/datum from D.
Suppose the table has r = |D| rows and c = |D| columns, the is said to be r \times c.

Communication Complexity

Communication complexity studies the number of bits of communication needed to compute a function in a game where the input is shared between the players. Typically the number of players is 2 unless specified oterwise.

In this model, the two players Alice and Bob are trying to compute a function f: X \times Y \to A. Alice gets an input x \in X and Bob gets an input y \in Y.

Asymmetric Communication

Miltersen~ considered a Communication game from a Static Data Structure Problem to prove Static Data Structure lower bounds for Polynomial Evaluation. This technique was subsequently distilled and used by Miltersen, Nisan, Safra, Wigderson~ to prove lower bounds for other data structures.

Note that the inputs of Alice and Bob in the Miltersen’s game are not of the same length and typically Alice’s input is much smaller than Bob’s input.

In such a scenario, keeping track of the total communication is not very useful as it is bounded by Alice’s input length since Alice can send her input to Bob and Bob can compute the function. However, the problem still remains interesting if we restrict Alice’s communication.

Therefore, it is natural to track the communication of Alice and Bob seperately in the Asymmetric setting.

We keep track of the number of bits Alice sends to Bob and the number of bits Bob sends to Alice seperately.

[!def] [a,b]-protocol An [a, b]-protocol for f is a protocol where the total number of bits that Alice sends Bob is at most a and the total number of bits that Bob sends Alice is at most b. ^d55d28

[!def] [t,a,b]^A-protocol A [t, a, b]^A-protocol for f is a protocol where each of Alice’s messages contains at most a bits and each of Bob’s messages contains at most b bits and at most t messages are sent, with Alice sending the first message. A [t, a, b]^B protocol is defined similarly. ^7fea2b

The richness method

[!def] (u,v)-richness A matrix M is (u, v)-rich if at least v columns contain at least u 1-entries. ^02c2c8 A communication problem f is (u,v)-rich if iff its Communication matrix M_f is (u,v)-rich.

[!lem] Lemma 5 Richness lemma Let f be a (u, v)-rich problem. If f has a deterministic [a, b]-protocol, then f contains a submatrix of dimensions at least u/2^{a+2} \times v^{a+b+2} containing only 1-entries. ^09a188

[!def] Skeleton The of a dynamic data-structure problem is a tuple (n, m, G), where n \in \N is the number of queries, m \in \N is the number of updates, and G is an update graph with m updates.

Given a skeleton S = (n, m, G), a with skeleton S, is a map T: V(G) \times [n] \to \ZO. For v \in V(G), we write q_i(v) = T(v,i). Then, a Boolean dynamic data-structure problem is a tuple P = (n, m, G, T), where (n,m,G) is a skeleton and T a table.

Let P = (n,m,G,T) be a dynamic data structure problem.

[!def] Dynamic Data Structure Scheme A (history-independent) , or , for P is a tuple A = (w,s,t_u, t_q, E,\{U_i\},\{Q_i\}), where: - E:V(G) \to (\ZO^w)^s is called the . - w \in \N is called the , and s \in \N is called the of the solution. - t_u \in \N is called the , and t_q \in \N the . - \{ Q_i \mid i \in [n] \} is a collection of decision-trees over (\ZO^w)^s, of depth \le t_q. - The decision-trees Q_i perform queries: forall v \in V(G), i \in [n], Q_i(E(v)) = q_i(v). - \{U_i \mid i \in [m]\} is a collection of read/write decision-trees over (\ZO^w)^s (i.e., over s-long words over a w-bit alphabet), of depth \le t_u. - The read-write decision-trees U_i perform updates on the encodings: for all v \in V(G), i \in [m], U_i(E(v)) = E(u_i(v)).

[!def] Relaxed Scheme A , or , is a tuple A = (w,s,t_u, t_q, E,\{Q_i\}), where: - w,s,t_u,t_q,E and \{Q_i\} are as items 1-5 above. - But the updates do not need to be efficiently implementable by read-write decision trees. Instead, we only require that, for every v \in V(G), the Hamming distance between E(v) and E(u_i(v)) is at most t_u. (This is always the case for a normal solution.)
